## 2. 总体设计判断

这套规范的气质不是消费型花哨视觉，而是偏企业工作台、数据可视化、业务操作系统：

- 品牌主色明确，以高饱和蓝色作为主驱动
- 中性色体系比较完整，强调“浅底、深字、弱边框”
- 组件状态覆盖完整，命名和变体管理很系统
- 信息层级偏理性，导航和按钮都很克制
- 适合业务管理、审批、运营、数据查看、任务处理类移动端场景

## 3. 色彩规范

### 3.1 品牌色

- `Brand-6 #0073E5`：常规主色
- `Brand-8 #004080`：点击态/更深一级主色
- `Brand-3 #80BFFF`：一般禁用
- `Brand-2 #B2D9FF`：文字禁用
- `Brand-1 #E6F2FF`：浅色底 / 浅悬浮底

结论：

- 主品牌色非常明确，是高识别蓝
- 悬浮、禁用、浅底不是另起一套颜色，而是围绕品牌蓝做明度递进
- 视觉上追求“统一、业务稳定、容易识别状态”

### 3.2 中性色

- `Line-1 #E3E5E8`
- `Fill-4 #D5D9DD`
- `Fill-3 #E3E5E8`
- `Fill-2 #F1F1F4`
- `Fill-1 #F7F7F8`
- `White #FFFFFF`

结论：

- 整体是非常浅的背景体系
- 边框存在感较弱，不依赖重边框分区
- 页面应该偏“白底 + 浅灰分层 + 深字”，而不是卡片厚重风

### 3.3 文字色

- `Text-5 #0B0D0F`：强调信息 / 正文标题
- `Text-4 #576675`：次强调 / 次级正文标题
- `Text-3 #8A99A8`：次要信息
- `Text-2 #C4CCD4`：置灰 / 禁用
- `Text-1 #FFFFFF`：白字

结论：

- 文字层级很清楚，适合高频信息扫描
- 标题与正文之间不是靠过多字号跳变，而是“字重 + 颜色 + 少量级差”来控层级

`圆角/直角, 按钮类型, 尺寸, 状态, 语义`

## 13. 输入框规范

### 13.1 总体结构

输入框页分两大类：

#### 水平型

- `水平,默认`
- `水平,输入`
- `水平,禁用`
- `水平,只读`
- `水平,错误`
- `水平,必填`

#### 垂直型

- `垂直,默认`
- `垂直,输入`
- `垂直,禁用`
- `垂直,只读`
- `垂直,错误`
- `垂直,必填`

此外还有组合输入组：

- `1` 到 `7` 组连续堆叠

说明这套输入体系既考虑单行字段，也考虑表单块级组合。

### 13.2 水平输入框

已读取示例：`水平,默认`

- 外层：`375 x 54`
- 实际内容容器宽：`359`
- 视觉为白底页面上的浅底输入区
- 标签：
  - `16px`
  - `#1C1F23`
- 占位文本：
  - `16px`
  - `rgba(28,31,35,0.35)`
- 底部有一条细分隔线

结论：

- 水平输入框更像“列表项里的输入单元”
- 不是传统厚边框输入框，而是信息列表式表单
- 很适合设置页、详情编辑页、审批字段页

### 13.3 水平错误态

已读取示例：`水平,错误`

- 外层仍是 `375 x 54`
- 标签仍为深色
- 错误内容变为：
  - `#EF2E1F`
  - `16px`
- 底部分隔仍保留

结论：

- 错误态通过文字颜色直接反馈，而不是重绘整块红框
- 这延续了整套规范“轻边框、轻容器、重文字状态”的策略

### 13.4 垂直输入框

已读取示例：`垂直,默认`

- 外层：`375 x 80`
- 内容容器宽：`359`
- 标签放上方：
  - `14px`
  - `#1C1F23`
  - 行高 `20`
- 输入内容放下方：
  - `16px`
  - 行高 `22`
  - 占位色为 `rgba(28,31,35,0.35)`
- 底部仍有细分隔

结论：

- 垂直输入框适合更长字段、更明确的标签说明
- 水平与垂直并存，说明这套规范支持不同信息密度场景

### 13.5 输入框设计判断

- 表单风格明显偏“列表式移动端表单”
- 分隔线比边框更重要
- 错误提示优先靠文字状态，而不是视觉震荡
- 这套输入框很适合映射成：
  - `layout: horizontal | vertical`
  - `state: default | typing | disabled | readonly | error | required`

## 17. 第二轮总结

这一轮可以确认几件对落地非常关键的事：

1. 雪花移动端规范的表单体系是“列表式表单”，不是厚边框 Web 表单。
2. 搜索、输入、选择器、对话框、轻提示都已经形成完整状态体系。
3. 反馈策略整体很克制，强调通过文字色、结构、层级来表达状态，而不是过度强调边框或装饰。
4. 这份规范很适合工程化沉淀成：
   - 颜色 token

1. **选择类组件不是孤立控件，而是完整场景组件**

4. **平台差异被保留在必要处**
   - 尤其是开关
   - 说明规范不是盲目统一，而是“统一品牌语言 + 尊重平台习惯”
