# Speckit UI 设计规范内置包

这份文件是 `speckit` 的内置 UI 规范包，用来统一承接原先分散在以下三处的参考：

- `AI_CRB_DESIGN_SPEC.md`
- `雪花移动端规范解读.md`
- `Snow-Design-Mobile-v2.1.1.figma.url`

从现在开始，`/speckit.ui` 默认优先使用这份内置包，而不是在执行时把三份资料当作外部依赖逐个引用。

## 1. 使用原则

- 这份包只保留对执行最重要的规则、风格判断和组件约束
- 如果后续规范更新，优先维护这份包，再决定是否回写源资料
- 视觉优先模式、HTML 交互网页输出、移动端与 PC 端适配，都以这份包为共同基础

## 2. PC 端规范摘要

适用场景：

- 管理后台
- 工作台
- 数据平台
- 运营系统
- 审批系统
- CRM / B 端业务系统

### 2.1 整体气质

- 主品牌色以 `#2C68FF` 为核心
- 大面积白底、浅灰底、弱描边、中等圆角
- 偏理性、可信、克制、高效，不走重装饰和过强氛围图形
- 适合长时间办公和高频信息扫描

### 2.2 信息密度

- 默认采用中高信息密度
- 正文字号以 `14px` 为基准
- 标题层级控制克制，不要堆很多超大标题
- 颜色主要用来区分主次、状态、操作，不做大面积装饰

### 2.3 色彩规则

- 品牌蓝主色：`#2C68FF`
- 主色 hover：`#1F54D9`
- 主色 active：`#1842AD`
- 页面背景：`#F7F9FD`
- 卡片背景：`#FFFFFF`
- 默认边框：`#DDE5F0`
- 主文本：`#0F172A`
- 次文本：`#475569`
- 弱文本：`#94A3B8`

功能色：

- 成功：`#16A34A`
- 警告：`#F59E0B`
- 危险：`#DC2626`

### 2.4 PC 端页面结构倾向

- 优先明确页面栅格、内容宽度、主次列关系
- 工作台页面要有清晰的首屏重心
- 列表、筛选、详情、表单、弹层之间要形成标准结构
- 按钮尺寸、表格可读性、导航与操作区关系要稳定

### 2.5 PC 端风格底线

- 不要做成营销页、海报页、氛围页
- 不要堆很多大色块、重阴影、夸张插画
- 不要把后台工具页做成低信息密度的展示图
- 优先考虑效率、判断、对比、录入、操作

## 3. 雪花移动端规范摘要

适用场景：

- 移动端业务操作页
- 任务处理页
- 培训 / 工作台 / 列表 / 表单 / 结果页

### 3.1 整体气质

- 主品牌蓝：`#0073E5`
- 点击态深蓝：`#004080`
- 页面整体是浅底、深字、弱边框
- 风格不是花哨消费视觉，而是理性、稳定、偏企业工作台

### 3.2 移动端颜色体系

- 品牌蓝是核心识别色
- 浅蓝底、浅灰底、中性色层级清晰
- 成功 / 警告 / 危险色都采用 1~5 档递进
- 状态应保持统一，不要为每个业务点随意起新色

### 3.3 字体和排版判断

- 标题与正文层级偏稳
- 导航标题常见为 `18px / Medium / 行高22`
- 按钮文字常见为 `16px / Medium / 行高22`
- 不依赖过度夸张的大字号营造层级

### 3.4 按钮系统

按钮变体维度：

- 形态：圆角 / 直角
- 类型：主要 / 辅助 / 描边
- 尺寸：特大 / 大 / 中 / 小
- 状态：默认 / 点击 / 禁用
- 语义：默认 / 危险 / 警告

常见按钮尺寸：

- `112 x 46`
- `96 x 38`
- `80 x 30`
- `64 x 24`

视觉特征：

- 主按钮：蓝底白字
- 辅助按钮：浅蓝底、蓝字
- 禁用态仍使用品牌蓝系递进

### 3.5 导航栏

- 标准导航栏主体：`375 x 44`
- 标题：`18px / Medium`
- 左侧返回图标：`24 x 24`
- 常见结构：纯标题 / 返回+标题 / 返回+顶部选项卡

### 3.6 搜索与输入

- 搜索框常见整体高度约 `56`
- 输入框分水平型与垂直型
- 错误态主要靠文案变红，不依赖整块红框
- 选择器常用底部弹起结构

### 3.7 选择与反馈组件

- 单选 / 复选通常优先作为列表项选择来组织
- 开关保留平台感，同时兼顾品牌化
- 日期选择器、区间选择、结果反馈、轻提示、弹层体系完整

### 3.8 移动端布局基准

- 默认以 `375 × 812` 作为主稿尺寸
- 要考虑安全区、滚动区、底部固定操作区
- 首屏要明确重心，不要把每个区块平均摊开

## 4. `/speckit.ui` 的统一执行约束

### 4.1 端类型优先规则

- 如果是移动端，默认先遵守本包中的移动端规则
- 如果是 PC 端，默认先遵守本包中的 PC 端规则
- 如果是双端，先分别建立两套约束，再考虑跨端统一

### 4.2 视觉优先模式下的共同原则

- 先问用户有没有参考图
- 先出单页成品级高保真方向图
- 用户确认方向后，再生成可点击交互的 HTML 网页效果稿
- 不能直接把第一张图机械翻译成代码

### 4.3 共同底线

- 规范用于收敛边界，不是把页面做回线框稿
- 不要牺牲视觉完成度去换取过早落地
- 不要脱离信息密度、交互效率和业务目标做装饰
